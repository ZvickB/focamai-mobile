# Mobile Project Notes

This folder holds the mobile-specific notes for the React Native app in `mobile/`.

## What belongs here
- Current mobile migration status
- Fast startup/reset notes for new chats
- Mobile-only implementation decisions and translation rules

## What stays in `../web/project-notes/`
- Canonical product behavior
- Backend reality
- Broader product direction
- AI/finalize strategy
- Business/compliance notes

## Startup read order
1. `../AGENTS.md`
2. `session-handoff.md`
3. `current-status.md`
4. `mobile-ui-ux-plan.md` when doing mobile UI/UX work
5. `restart-strategy.md`
6. `../migration-roadmap.md`
7. `../CLAUDE.md`
8. `../../web/project-notes/app_flow.md`
9. `../../web/project-notes/doc_briefs.md`

## UI/UX planning priority
- `mobile-ui-ux-plan.md` is now the primary implementation-order doc for mobile UI/UX work.
- `mobile-ui-ux-direction.md` remains the principles and taste reference.
- `proposed-layout.md` and the PNG mockups are brainstorming inputs only, not final screen specs.

## Working rule
- Most product thinking happens in `web/`.
- Mobile notes should explain how that product is being translated into React Native right now.
